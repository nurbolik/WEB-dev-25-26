let message; // Declaration of a variable message = 'Hello!'; // store the string 'Hello' in the variable named message alert(message); // shows the variable content let message = 'Hello!'; // define the variable and assign the value alert(message); // shows the value: 'Hello!' let user = 'John', age = 25, message = 'Hello'; // Declare multiple variables in one line let user = 'John', age = 25, message = 'Hello'; Может показаться, что так короче, но мы не рекомендуем так делать. Для удобства чтения используйте по одной строке на переменную. Многострочный вариант немного длиннее, но его легче читать: let user = 'John'; let age = 25; let message = 'Hello'; Некоторые также определяют несколько переменных в таком многострочном стиле: let user = 'John', age = 25, message = 'Hello'; ...Или даже в стиле «сначала запятая»: let user = 'John' , age = 25 , message = 'Hello'; let $ = 1; // declared a variable with the name "$" let _ = 2; // and now a variable with the name "_" alert($ + _); // 3 Примеры неправильных названий переменных: let 1a; // cannot start with a digit let my-name; // hyphens '-' aren't allowed in the name Дело имеет значение Переменные с именами apple и APPLE — это две разные переменные. Использование нелатинских букв допускается, но не рекомендуется Можно использовать любой язык, в том числе кириллицу, китайские иероглифы и так далее, например: let имя = '...'; let 我 = '...'; Технически здесь нет ошибки. Такие имена допустимы, но существует международное соглашение об использовании английского языка в именах переменных. Даже если мы пишем небольшой скрипт, он может прослужить нам долго. Возможно, людям из других стран когда-нибудь понадобится его прочитать. Зарезервированные имена Существует список зарезервированных слов, которые нельзя использовать в качестве имен переменных, поскольку они используются самим языком. Например: let, class, return и function зарезервированы. Приведенный ниже код выдает синтаксическую ошибку: let let = 5; // can't name a variable "let", error! let return = 5; // also can't name it "return", error! Задание без use strict Обычно нам нужно определить переменную перед ее использованием. Но в старые времена было технически возможно создать переменную простым присвоением значения без использования let. Это все еще работает сейчас, если мы не добавим use strict в наши скрипты для поддержания совместимости со старыми скриптами. // note: no "use strict" in this example num = 5; // the variable "num" is created if it didn't exist alert(num); // 5 Это плохая практика, которая приведет к ошибке в строгом режиме: "use strict"; num = 5; // error: num is not defined Константы Чтобы объявить константную (неизменяемую) переменную, используйте const вместо let: const myBirthday = '18.04.1982'; Переменные, объявленные с помощью const, называются «константами». Их нельзя переназначить. Попытка сделать это приведёт к ошибке: const myBirthday = '18.04.1982'; myBirthday = '01.01.2001'; // error, can't reassign the constant! Если программист уверен, что переменная никогда не изменится, он может объявить её с помощью const, чтобы гарантировать это и сообщить об этом всем. Константы в верхнем регистре Широко распространена практика использования констант в качестве псевдонимов для труднозапоминаемых значений, которые известны до выполнения программы. Такие константы называются с помощью заглавных букв и символов подчеркивания. Например, давайте создадим константы для цветов в так называемом «веб-формате» (шестнадцатеричном): const COLOR_RED = "#F00"; const COLOR_GREEN = "#0F0"; const COLOR_BLUE = "#00F"; const COLOR_ORANGE = "#FF7F00"; // ...when we need to pick a color let color = COLOR_ORANGE; alert(color); // #FF7F00 Преимущества: COLOR_ORANGE Его гораздо проще запомнить, чем "#FF7F00". Его гораздо проще перепутать с "#FF7F00", чем с COLOR_ORANGE. При чтении кода COLOR_ORANGE гораздо понятнее, чем #FF7F00. Когда следует использовать заглавные буквы для обозначения константы, а когда — нет? Давайте проясним этот вопрос. «Константа» означает, что значение переменной никогда не меняется. Но некоторые константы известны до начала выполнения программы (например, шестнадцатеричное значение красного цвета), а некоторые константы вычисляются во время выполнения программы, но не меняются после первоначального присвоения. Например: const pageLoadTime = /* time taken by a webpage to load */; Значение pageLoadTime неизвестно до загрузки страницы, поэтому оно называется обычным. Но это всё равно константа, потому что она не меняется после присваивания. Другими словами, константы с заглавной буквы используются только в качестве псевдонимов для «жестко заданных» значений. Правильно называйте вещи Если говорить о переменных, то есть еще один чрезвычайно важный момент. Имя переменной должно иметь четкое и понятное значение, описывающее данные, которые она хранит. Присваивание имён переменным — один из самых важных и сложных навыков в программировании. Достаточно взглянуть на имена переменных, чтобы понять, какой код написан новичком, а какой — опытным разработчиком. В реальном проекте большая часть времени уходит на модификацию и расширение существующей кодовой базы, а не на написание чего-то с нуля. Когда мы возвращаемся к какому-то коду после того, как какое-то время занимались чем-то другим, нам гораздо проще найти нужную информацию, если она хорошо структурирована. Другими словами, если у переменных хорошие названия. Пожалуйста, потратьте время на то, чтобы придумать правильное название для переменной, прежде чем объявлять ее. Это окупится сторицей. Вот несколько полезных правил: Используйте удобочитаемые имена, например userName или shoppingCart. Избегайте сокращений или коротких имён, таких как a, b, и c, если только вы не знаете, что делаете. Имена должны быть максимально информативными и лаконичными. Примеры неудачных имён: data и value. Такие имена ничего не говорят. Их можно использовать только в том случае, если из контекста кода совершенно очевидно, на какие данные или значения ссылается переменная. Договоритесь об обозначениях внутри команды и в своей голове. Если посетитель сайта называется «пользователем», то соответствующие переменные следует называть currentUser или newUser вместо currentVisitor или newManInTown. Звучит просто? Так и есть, но на практике создавать информативные и лаконичные имена переменных не так просто. Дерзайте. Повторно использовать или создавать? И последнее замечание. Некоторые ленивые программисты вместо того, чтобы объявлять новые переменные, используют уже существующие. В результате их переменные похожи на коробки, в которые люди складывают разные вещи, не меняя наклеек. Что сейчас в коробке? Кто знает? Нужно подойти поближе и проверить. Такие программисты немного экономят на объявлении переменных, но в десять раз больше теряют на отладке. Дополнительная переменная — это хорошо, а не плохо. Современные минификаторы JavaScript и браузеры достаточно хорошо оптимизируют код, поэтому проблем с производительностью не возникнет. Использование разных переменных для разных значений может даже помочь движку оптимизировать ваш код. Краткие сведения Мы можем объявлять переменные для хранения данных, используя ключевые слова var, let или const. let – это современное объявление переменной. var – это объявление переменной старой школы. Обычно мы его вообще не используем, но мы рассмотрим тонкие отличия от let в главе Старого "var", на всякий случай, если они вам понадобятся. const – похоже на let, но значение переменной не может быть изменено. Имена переменных должны быть такими, чтобы мы могли легко понять, что в них содержится.
